## 何时触发

★ 判断标准: **用户是否要一份完整的旅行规划网页（HTML）**

需要 delegate:
  - "帮我规划北京3日游"
  - "生成一份杭州旅行攻略"
  - "下周五去成都4天，做个行程"
  （这些是重任务：天气 + 多轮 POI + 酒店 + 小红书 = 数十次工具调用，
   raven 自己做会阻塞 5-10 分钟且质量不可靠）

**不需要** delegate:
  - "杭州有什么好吃的？"         → 普通问答，自己搜一下文字回复
  - "推荐几个西湖边的景点"        → 普通问答
  - "北京天气怎么样"              → 普通问答
  （这些不需要 HTML，直接文字回答即可）

## 工作流（一个路径）

```
1. 确认目的地、日期、偏好
   - 如果用户没给全，追问补齐（城市、起止日期、旅行偏好）
   - 不要问太多轮，基本信息够了就提交

2. 立即委托
   - 调 delegate_travel_planner(goal="<用户原始请求>")
   - 告知用户: "规划已提交，3-5分钟后通知您。您可以继续聊其他话题。"

3. 等待系统通知（Phase 3 会自触发，不需要你记）
   - ★ 系统通知中已包含 JSON 文件路径和 render_trip_html 调用指令
   - 你不需要回忆这条 SKILL——通知文本会直接告诉你该做什么
```

## 关于 render_trip_html

同一份 JSON 可无限次重渲染不同风格——换风格只需调 render_trip_html 并传入 style 参数，不需要重新规划。
